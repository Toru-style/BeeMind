🧩 Tetu Log_now v.1.0
採用日：2025年06月22日（VN 04:30）
改訂日：

1. 今のTetuについて
今のTetuは、オフショアIT企業の社長として、ベトナムを拠点に仕事をしている。
思考・記録・行動のスタイルは、誰かの模倣ではなく、自分なりに辿り着いた、積み重ねの結果。
日々考えていることは、何か特別な思想を生み出すためではない。
ただ「常に考えてしまう性格」で、気づけば複数の可能性やパターンが自然に脳内に広がっているだけ。
だからこそ、「自己と問いの連続」こそが自分の本質だと思っている。
自分でも「ただの捻くれ者」と思うことがある。
でも、出発点はいつもそこだった。

2. 思考と会社
会社の仕組みは、「シンプル」、「極力無駄を省く」、「誰でも回せる設計」が核。
主に使っているのは、Excel（スプレッドシート）・チャットワーク・Backlogの3つだけ。
基本は「2ヶ月で独り立ち」、しかも残業ほぼゼロ。
その思想と近い存在がBee Mind。
人間の選択や思考の再現性を、無料・スマホ完結で仕組み化したもの。
ただし、Bee MindとTetu個人の話は切り離して記録していく方針。あくまでの実績、一貫性の一つとして紹介。

3. 自分の性格と特性
後咲きタイプ：「最初は遅いが、理解と実行に入ると爆速で進む」
「成長障害かと思った」こともあったが、実際は急加速型で意外性が高い
周囲からは「早すぎ」「怖っ」と言われることも
「効率化最幸」主義：無駄が嫌い、使いにくいものが嫌い。
「使いたいものだけ使う」「やりたいことだけやる」──シンプル追加型の思考。

4. 組織再構築と社員たち
Tetu赴任前の会社はもともと赤字、案件も崩壊、体制不明。
Tetuがゼロから仕組みを構築し、1年で黒字化。
外的要因で拡大 → 再縮小を経ても、残った社員と新体制で立て直した。
いまの社員たちは、「会社がやばい時に残ってくれた仲間たち」。
給料を下げた時も、前を向いて一緒に頑張ってくれた。
ベトナムでこの関係性は異例だと感じている。

5. 発信と記録への思い
本当はずっと記録してきた。でも、発信するタイミングと、続ける自信がなかった。
いまようやく、発信できるタイミングに来たと感じている。
シリーズ3まではBee Mind、そのあとは自分自身の話や裏話を発信予定。

6. タイトルに込めた想い
全体のタイトルは
『ベトナムからの感謝』。
自分がこうして発信できているのは、「ベトナムに来たから」だと思っている。
すべての始まりは、ここで覚悟が決まり、家族や仲間と出会えたこと。
今ここに立てているのは、自分1人の力ではない。
出会ってきた人たちがいて、支えてくれた人たちがいて、ベトナムでチャンスを得て、掴み取り、形にしたから。

🧩 締めの言葉
Tetuのやってきたことは、誰よりも「基本」に忠実で、
誰よりも「誰かのため」「何かのため」に徹してきた結果。
偉くなりたいわけじゃない、ただ「一緒にやってくれる人たちに応えたい」。
その芯の強さが、Bee Mindにも、会社にも、思考にも、滲み出ている。

🧩 Tetu Log_then v.1.0
採用日：2025年06月22日（VN 04:50）
改訂日：

1. 出発点と価値観の芽生え
Tetuはもともと文系出身で、大学までパソコンに触れた経験もほとんどなかった。
文系出身 → IT業界SE → IT営業 → BrSE → IT営業 → ベトナムIT。
社会人から急にIT企業へ就職し、新世界へ進んだ。社会人になってから、特にExcelを使い始め、18年近く使い続けてきた。
形式や出力はパターン化され、シンプルな構造と整理欲求が強い性格。
自分にとっての最適を優先し、テストや再現性を重視するスタンスは昔から変わらない。

2. 技術との関わり
社会人の最初はSE（システムエンジニア）として6年半働いていた。主にPM系の役割で、使用言語はCOBOLやVBなど基幹系が中心。
社会人以降にIT技術を吸収しており、「理屈よりも実証」「とにかくやってみる」が行動スタイルだった。
Excelは、関数・見栄え・ピボット・VBAといった機能に加え、
スプレッドシート＋GASを用いた簡易DB的運用へと進化させていった。

3. 転機と行動の背景
2018年：離婚を経験。
2019年：本格的に日本を離れ、拠点を変更。英語はその時点で話せなかった。
2020年：再度ベトナムに渡り、ある会社の手伝いから再出発。半年で黒字化。
2021年：社長就任。ベトナム人女性と再婚。
2022年：MA（M&A）。
2023年：親会社の影響で赤字化 → MBOで再度独立。
2024年：再建順調。Bee Mind構想を開始。
2025年：note配信やBee Log構想を本格スタート。

4. 社会や他者との関係
組織再建や社員との関係において、Tetuは「自分の強さ」よりも「一緒にやる人たちとの信頼」を大事にしてきた。
赴任前の旧会社体制の崩壊から、仕組みをゼロから構築し、赤字から黒字化、社員の定着、文化の再設計まで、一貫して自ら手を動かしてきた。
外的要因による拡大・縮小を経ても、残った仲間と共に「再出発」を繰り返す中で、
「やってきたことは全部“たまたま”だった」と語るが、そのたまたまの蓄積が今に繋がっている。

5. 記録と気づきの蓄積
Tetuは、これまで何度も「発信」を考えてきたが、続ける自信がなかった。
でも実際には、ずっと記録を続けてきた。
ChatGPTを使い始めたきっかけは、GASの質問と「カジノはほとんど負ける？」という雑談から。
それが「人生≒ギャンブルのようだ」という気づきにつながり、
やがて「記録すること」「仕組みに作ること」に発展していく。

6. 想いと出発点
人生をギャンブルのように捉えるようになったのは40歳ごろ。
流れやパターンを感じるようになり、「たまたま」を繋げて意味を見出すようになった。
「偉くなりたい」とは思っていない。
ただ「一緒にやってくれる人に応えたい」。
その思いが、過去の行動にも、仕組みにも、全てににじみ出ていた。

🧩 締めの言葉（Tetu_nowとの対比）
Tetu_nowは、「今」を言葉にしたもの。
Tetu_thenは、「今」に至るまでに、Tetuが何を見て、何を積み上げてきたかの記録。
過去を振り返るのは、自慢や後悔のためではない。
今の実績、今までの実績が、「誰かの何かの役に立つはず」「何かのきっかけになるはず」と考えた結果。

🧩 Tetu Log_core v1.1
採用日：2025年06月22日（VN 05:30）
改訂日：2025年06月24日（VN 18:30）

1. 記録全体の意図と背景
Tetuの記録は、過去の整理や自慢ではなく、
「これまで積み重ねてきたことが、誰かの気づきやきっかけになるかもしれない」
という想いから始まった。
自分の言葉で、自分の歩みを記録する。
補完も美化もせず、ただの事実と構造を積み重ねていく。
そうした「記録の土台」として、このTetu_coreは存在している。

2. 各記録ファイルの意味と役割
Tetu_now：現在のTetuを構成する考え方・姿勢・状態の明文化
Tetu_then：これまで歩んできた背景・行動・転機の記録
Tetu_stack：知識・スキル・習慣など、積層された実力と特性の可視化
それぞれのファイルは、重複を避け、特化した役割で構成されている。
この core が全体構造を支える中核となり、個別ファイルはその枝葉として機能する。

3. なぜこの構造にしたのか
記録を残す上で最も重視したのは 「わかりやすさと再利用性」。
どんな状態でも迷わずに振り返れる構造にしておきたい。
now / then / stack という分類は、👉「今」「過去」「積み重ね」に自然に分類できるため採用。
core は、それらを支える 「設計思想」「整理思想」 の起点。
記録の再現性・拡張性を確保するために、余計な演出や枝葉末節は削ぎ落とし、
誰が読んでも、“Tetuという人物の思考が辿れる”ような構造を志向している。

4. Bee Mindとの関係と切り分け
Bee Mind は、Tetuの作業支援を目的とした「AI補助OS」構想であり、
この Tetu_core とは明確に切り分けられている。
Tetu_core は「Tetuの核」
Bee Mind は「Tetuを支える思想ツール」
記録・構造として交差する部分はあるが、
「Tetu自身の言葉で語る記録」は、この core 系列に集約していく方針。

5. 今後の構想と整理方針
今後、以下のファイルが必要になった場合も、core の下位ファイルとして扱う：
Tetu_curve：変化・成長・揺らぎの履歴
Tetu_build：行動・構築・仕組み化の実例集
Tetu_voice：未分類・発信ログ・肉声に近い記録など
ただし現時点では作成を保留し、必要になった場合に限り追加。
core → now / then / stack が完成しているため、無理に増やさないことを前提とする。

6. 発信・保存方針
noteシリーズ4以降の発信ベースとして使用予定
Mdファイルは使用せず、テキストベースでの出力・保存をTetuが手動で行う
ChatGPTの補完・推測は禁止とし、Tetuの言葉・記録に限定する
必要に応じて、coreからログ化（Bee Log系）への転記も可能

7. 締めの言葉
Tetu_coreは、「今のTetuを支えている全体像」を言葉にしたもの。
すべての発信、すべての思考、すべての行動がここに繋がってくる。
だからこそ、これが思考記録の原点であり、今後すべての記録の基盤となる。
