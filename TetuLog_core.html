1. 記録全体の意図と背景
Tetuの記録は、過去の整理や自慢ではなく、
「これまで積み重ねてきたことが、誰かの気づきやきっかけになるかもしれない」
という想いから始まった。
自分の言葉で、自分の歩みを記録する。
補完も美化もせず、ただの事実と構造を積み重ねていく。
そうした「記録の土台」として、このTetu_coreは存在している。

2. 各記録ファイルの意味と役割
Tetu_now：現在のTetuを構成する考え方・姿勢・状態の明文化
Tetu_then：これまで歩んできた背景・行動・転機の記録
Tetu_stack：知識・スキル・習慣など、積層された実力と特性の可視化
それぞれのファイルは、重複を避け、特化した役割で構成されている。
この core が全体構造を支える中核となり、個別ファイルはその枝葉として機能する。

3. なぜこの構造にしたのか
記録を残す上で最も重視したのは 「わかりやすさと再利用性」。
どんな状態でも迷わずに振り返れる構造にしておきたい。
now / then / stack という分類は、👉「今」「過去」「積み重ね」に自然に分類できるため採用。
core は、それらを支える 「設計思想」「整理思想」 の起点。
記録の再現性・拡張性を確保するために、余計な演出や枝葉末節は削ぎ落とし、
誰が読んでも、“Tetuという人物の思考が辿れる”ような構造を志向している。

4. Bee Mindとの関係と切り分け
Bee Mind は、Tetuの作業支援を目的とした「AI補助OS」構想であり、
この Tetu_core とは明確に切り分けられている。
Tetu_core は「Tetuの核」
Bee Mind は「Tetuを支える思想ツール」
記録・構造として交差する部分はあるが、
「Tetu自身の言葉で語る記録」は、この core 系列に集約していく方針。

5. 今後の構想と整理方針
今後、以下のファイルが必要になった場合も、core の下位ファイルとして扱う：
Tetu_curve：変化・成長・揺らぎの履歴
Tetu_build：行動・構築・仕組み化の実例集
Tetu_voice：未分類・発信ログ・肉声に近い記録など
ただし現時点では作成を保留し、必要になった場合に限り追加。
core → now / then / stack が完成しているため、無理に増やさないことを前提とする。

6. 発信・保存方針
noteシリーズ4以降の発信ベースとして使用予定
Mdファイルは使用せず、テキストベースでの出力・保存をTetuが手動で行う
ChatGPTの補完・推測は禁止とし、Tetuの言葉・記録に限定する
必要に応じて、coreからログ化（Bee Log系）への転記も可能

7. 締めの言葉
Tetu_coreは、「今のTetuを支えている全体像」を言葉にしたもの。
すべての発信、すべての思考、すべての行動がここに繋がってくる。
だからこそ、これが思考記録の原点であり、今後すべての記録の基盤となる。
